Fork

  • 打开一个他人的项目，点击 Fork，项目就会拷贝到你的账号下。
  • 克隆到本地：git clone <项目地址>

    注意，这里的项目地址要使用 SSH 而不是 HTTPS。否则后面 git push 的时候会报错提示账号密码提交已被废弃。

  • Fork 与 upstream 保持一致。虽然这不是必要的一步，但是如果你想参与更多而不是仅仅做一个 quick fix, 你需要保证你的代码与远程原始代码库 (the original “upstream” repo) 保持一致。做到这一步，你仅需要设置下 remote:
    # Add 'upstream' repo to list of remotes
git remote add upstream https://github.com/UPSTREAM-USER/ORIGINAL-PROJECT.git

# Verify the new remote named 'upstream'
git remote -v
  • 创建并切换到新分支：git checkout -b sl-patch
  • 修改项目，并提交：
    git add -A
git commit -m <修改说明>
  • 推送
    git push origin sl-patch
  • Pull Request
  • 提交后等待作者合并，合并完成即可删除分支。
  • 下次有新的修改时，要先合并上游更新到本地：
    git pull --rebase upstream source

问题

由于换了新电脑，git 需要重新配置，否则远程仓库无法识别 ssh 密钥。

git config --global user.name "your_name"
git config --global user.email "your_email"

// 创建密钥，使用 ed25519 加密
ssh-keygen -t ed25519 -C <your_email>

// ssh 默认验证列表中并没有 id_ed25519，只有 id_rsa，所以需要进行添加
ssh -v git@github.com
ssh-agent -s
ssh-add ~/.ssh/id_ed25519
// ssh-add 后可能会出现重启电脑后失效的问题，将其添加到 .zshrc 配置文件可解决。

// 验证
ssh -T git@github.com

最后在 Github 添加新的 SSH Key 即可。
