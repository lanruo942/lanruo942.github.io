Joomla 安装 & 修改 phpmywind flash 文件上传

Joomla phpmywind

Joomla 安装第三步确认配置时，PHP配置中会建议关闭 输出缓冲（output_buffering），事实上保持默认的开启状态（4096 bytes）即可。
参考：深入理解php的输出缓冲区(output buffer)

体验

Joomla 和 Drupal都依赖于 .htaccess 文件来进行链接重定向，所以最好是使用Apache，免去配置Nginx的麻烦（Nginx默认配置不读取 .htaccess 文件，会导致 Joomla 前台菜单链接 404、Drupal 后台菜单链接 404）。

Joomla 体验下来发现在整合前端代码的时候非常依赖后台的模块，这样就需要把前端代码拆散拼入后台，会导致前后端难以分离。而且创建模板门槛也比较高，需要专门的模板架构进行模板开发（如T3、Helix、Gantry、Wrap、EF4、BootstrapJoomla 等），所以更适合大型网站使用，对中小型网站来说过于复杂，尤其是国内客户看到 Joomla 的后台可能会非常头大。

更改phpmywind的文件上传模块

phpmywind用的文件上传是uploadify.js（Flash版本），同款还有个html5版本uploadifive.js。两款上传插件都多年不维护了，对比发现两者的API并不通用。

先确定phpmywind后台上传模块的位置：/admin/plugin/uploadify

需要修改的是 index.php 文件，Flash版本用到swf模块可以忽略掉。下载uploadifive源文件，将其中的 jquery.uploadifive.min.js拷贝到当前目录，uploadifive.php和uploadifive.css用不到，还是用原来的uploadify.php和uploadify.css。

修改 index.php 文件：

  • jquery.uploadify.min.js 改为 jquery.uploadifive.min.js
  • $('#uploadify').uploadify 改为 $('#uploadify').uploadifive
  • onUploadStart改为 onUploadFile
  • onUploadSuccess 改为 onUploadComplete
  • 调用函数中新增两条：
    • 'uploadScript': 'uploadify.php',
    • 'buttonText': '选择文件',

Done.

修改 5.6 版本的上传模块后，自定义模型中的自定义变量上传文件时会出现 多个媒体文件 保存后不显示的问题。原因是上传模块中 变量 被赋值 string 后又赋值 array，Error: [] operator not supported for strings

// 修改 admin/modeldata_save.php, 将受影响的 line 95/233 forEach 前添加 
$v = [];

// 131/273 行 for 循环前的 $vTmp = '' 改为
$vTmp = [];
